# حساب و کتاب من — اپ اندروید حسابداری مخارج شخصی (فارسی)

اپلیکیشن اندروید بومی با Kotlin + Jetpack Compose برای ثبت و مدیریت مخارج شخصی به ریال، با تقویم هجری شمسی.

## امکانات

- ثبت هزینه با تاریخ شمسی (تقویم اختصاصی، بدون هیچ کتابخانه‌ی خارجی)
- تفکیک اختیاری هر هزینه بر اساس حساب بانکی یا نقدی
- شناسایی خودکار مبلغ از پیامک بانک (با صف «در انتظار تایید» برای جلوگیری از ثبت اشتباه)
- گزارش و نمودار دونات بر اساس روز/هفته/ماه/سال، گروه‌بندی بر اساس دسته‌بندی یا حساب
- خروجی اکسل واقعی (.xlsx) چهار-شیتی: لیست کامل، خلاصه دسته‌بندی، خلاصه حساب، خلاصه بازه زمانی
- بکاپ/بازیابی کامل داده‌ها با فایل JSON قابل اشتراک‌گذاری (گوگل‌درایو، تلگرام و ...)
- رابط کاربری فارسی/راست‌به‌چپ با رنگ‌های پاستیلی هماهنگ و فونت وزیرمتن
- جداکننده‌ی سه‌رقمی اعداد + ارقام فارسی در سراسر اپ

## قبل از اجرا — این چند قدم ضروری است

1. **فونت‌ها را اضافه کنید**: فایل‌های ttf وزیرمتن را در مسیر `app/src/main/res/font/` با نام‌های
   `vazirmatn_light.ttf`, `vazirmatn_regular.ttf`, `vazirmatn_medium.ttf`, `vazirmatn_semibold.ttf`, `vazirmatn_bold.ttf`
   قرار دهید (توضیح کامل در `app/src/main/res/font/README.txt`). **بدون این فایل‌ها پروژه build نمی‌شود.**
2. پروژه را در Android Studio (نسخه‌ی جدید، Kotlin 1.9.24 / AGP 8.4.1) باز کنید و Gradle sync بزنید. Gradle wrapper
   (نسخه‌ی ۸.۷) از قبل داخل پروژه هست، پس نیازی به نصب دستی Gradle نیست.
3. minSdk = 26 (اندروید ۸ به بعد).

## ساخت خودکار APK با GitHub Actions

دو workflow آماده در `.github/workflows/` قرار دارد:

- **`build-apk.yml`**: با هر push یا از تب Actions (اجرای دستی)، یک APK دیباگ و یک APK ریلیز بدون امضا می‌سازد
  و به‌عنوان Artifact قابل دانلود در همان صفحه‌ی اجرای workflow می‌گذارد. نیازی به هیچ تنظیم اضافه‌ای نیست؛
  فقط باید فونت‌ها را قبل از push کردن در `app/src/main/res/font/` قرار داده باشید (چون در .gitignore مستثنا نشده‌اند).
- **`release.yml`**: با push کردن یک تگ نسخه (مثل `v1.0.0`) یک APK **امضاشده** می‌سازد و به‌صورت خودکار یک
  GitHub Release با همان APK ضمیمه می‌سازد. برای این یکی باید ۴ سیکرت را در تنظیمات ریپازیتوری اضافه کنید —
  راهنمای کامل (شامل دستور ساخت keystore) بالای همان فایل workflow نوشته شده.

## نکته‌ی مهم درباره‌ی مجوز پیامک و انتشار در Play Store

گوگل‌پلی از سال ۲۰۱۹ دسترسی به `READ_SMS`/`RECEIVE_SMS` را محدود کرده و فقط به اپ‌های «مدیریت مالی شخصی» با یک
اعلامیه‌ی رسمی (Permissions Declaration Form) در Play Console اجازه می‌دهد. برای استفاده‌ی شخصی/نصب مستقیم (sideload)
هیچ محدودیتی نیست، اما اگر قصد انتشار در Play Store را دارید، باید این فرم را قبل از انتشار پر کنید؛ در غیر این صورت
اپ رد می‌شود.

## معماری

- **UI**: Jetpack Compose + Material 3، ناوبری با Navigation-Compose
- **دیتابیس**: Room (SQLite) — سه جدول: `expenses`, `categories`, `accounts`
- **تقویم شمسی**: `util/PersianDateUtil.kt` — پیاده‌سازی مستقل بر پایه‌ی الگوریتم استاندارد jalaali +
  الگوریتم Fliegel–Van Flandern برای میلادی↔JDN. **با ۷۶٬۲۴۸ آزمون خودکار روی تمام سال‌های ۱ تا ۳۱۷۷ شمسی
  اعتبارسنجی شده** (رفت‌وبرگشت تاریخ بدون خطا).
- **پیامک**: `sms/SmsReceiver.kt` + `sms/SmsParser.kt` — Regex برای استخراج مبلغ و ۴ رقم آخر کارت
- **اکسل**: `util/XlsxWriter.kt` — نویسنده‌ی دستی و سبک فرمت xlsx (بدون Apache POI، چون POI روی اندروید
  معمولاً مشکل بیلد/حجم ایجاد می‌کند). خروجی با اکسل، گوگل‌شیت و لیبره‌آفیس تست شده (اعتبارسنجی با openpyxl).
- **بکاپ**: `util/BackupManager.kt` — سریالایز JSON با `org.json` (built-in، بدون وابستگی اضافه)

## محدودیت مهم این تحویل

این پروژه در یک محیط sandbox بدون Android SDK/Gradle/امولاتور نوشته شده، پس **مستقیماً build و اجرا نشده** —
اما موارد زیر به‌صورت مجزا با تست خودکار (پایتون، معادل منطق Kotlin) اعتبارسنجی شدند:
- الگوریتم تبدیل تاریخ شمسی↔میلادی (۷۶ هزار مورد، بدون خطا) — یک باگ واقعی در این بخش پیدا و برطرف شد
- محاسبه‌ی روز هفته (اصلاح شد، آفست اشتباه بود)
- نویسنده‌ی XLSX (فایل واقعی ساخته و با openpyxl باز و بررسی شد)
- Regex استخراج مبلغ از پیامک بانکی (روی چند نمونه‌ی واقعی پیامک بانک تست شد)

توصیه می‌کنم بعد از افزودن فونت‌ها، یک‌بار `./gradlew assembleDebug` را اجرا کنید و اگر خطای کامپایلی دیدید
(بعید است، ولی محیط واقعی Android Studio دقیق‌تر از بازبینی دستی است) به من بگویید تا برطرف کنم.
